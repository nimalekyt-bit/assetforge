# 03 — Умные фичи (что сделает AssetForge «умным»)

Ключевое слово задачи — **УМНЫЙ**. Ниже три пласта интеллекта: «понимание» (что загрузили и как это
обработать), «помощник» (UX, который ведёт пользователя) и «продвинутая обработка» (качество выреза,
эффекты, форматы). Большинство фич — чистый Python+Pillow+numpy; тяжёлый AI — опционально с фолбэком.

> Принцип: пользователь говорит «куда» (площадка/формат), а не «как» (ползунки). Инструмент сам решает,
> как кадрировать, масштабировать и на какой фон класть контент — и предупреждает, если что-то не так.

## Идеи «ума» из ресёрча (что делают топовые сервисы)

- **Subject-aware/saliency smart-crop** под любой aspect ratio (Cloudinary g_auto, Twitter, Azure, AEM):
  фокальная точка считается один раз и переиспользуется для всех размеров. Базовая saliency реализуема
  на чистом numpy (спектральный остаток/градиенты). **Урок индустрии: всегда давать ручной override.**
- **Авто-классификация типа ассета** (logo/photo/icon/screenshot) → авто-рекомендация пресета.
  Эвристики (аспект, доля альфы, число цветов, резкость краёв) дают ~70-80% пользы почти даром.
- **AI-матирование волос/меха/стекла** (BiRefNet/isnet через rembg-ONNX) — главный скачок «ума» восприятия.
- **AI-апскейл** (Real-ESRGAN ONNX) — реально новые детали вместо интерполяции; авто-триггер из QA.
- **Извлечение доминирующего цвета/палитры** — для favicon-фона, `theme_color` манифеста, превью.
- **Авто-коррекция тона** (autocontrast, gray-world WB, CLAHE) — тусклый исходник чистится сам.
- **QA-отчёт** (блюр через variance of Laplacian, разрешение vs цель, выход за safe-area, контраст).
- **Замена/генерация фона** + синтез тени (drop/contact) — объект не «висит» на новом фоне.
- **Device-mockup превью и favicon-checker** — видно финальный контекст и что иконка не обрежется.

## A. Умный движок обработки

| Фича | Польза | Сложность | Эффект |
|------|--------|:---------:|:------:|
| Авто-классификатор типа ассета + рекомендация пресета | Снимает «загрузил, куда кликать?» | M | высокий |
| Saliency/subject-aware кроп под несквадратные цели | Контент не режется под баннер/аватар | L | высокий |
| QA-движок предупреждений о качестве | Предупреждает до экспорта, а не выдаёт мусор молча | M | высокий |
| Edge-preserving / опц.-AI апскейл | Мелкий исходник → крупный размер без мыла | L | средний |
| Авто color/contrast/levels препроцессинг | Тусклый/жёлтый исходник чистится сам | M | средний |
| Извлечение палитры и доминирующего цвета | Данные для манифестов и брендовых превью | M | средний |
| Адаптивный авто-детект фона (неравномерное освещение) | Авто-режим работает на реальных фото | L | средний |
| Авто-выбор параметров кропа/выравнивания по типу | Вордмарку — воздух, иконке — квадрат, фото — центр масс | M | средний |

**Авто-классификатор + рекомендация пресета.** `core/classify.py`: эвристики numpy/Pillow без сети —
aspect bbox (>2.2 → wordmark/logo, ~1:1 → icon), число уникальных цветов (мало + резкие края →
vector-like, много + градиенты → photo), наличие исходной альфы, `objects>3` → sprite-sheet. Маппинг
`kind→preset` таблицей; в `meta`: `asset_kind`, `suggested_preset`, `reasons[]`. Подключить в `analyze()`,
вернуть в `/api/analyze`, показать бейдж «Похоже на логотип» + кнопку «Применить рекомендованный».

**Saliency-кроп.** `core/saliency.py`: карта на чистом numpy = alpha-маска + |Sobel|-градиенты
(+ опц. spectral-residual FFT). `saliency_center(arr)->(cx,cy)`. В `render_target`/`crop` окно нужного
соотношения максимизирует сумму значимости (по интегральному изображению, O(W·H)). `focus=[fx,fy]` для
ручного override; деградирует к центру при равномерной значимости.

**QA-движок.** `core/qa.py`: `qa_report(...)->list[QAWarning{code,severity,message,hint,fix}]`.
Проверки переиспользуют `tests/metrics.py`: `UPSCALE_RISK` (объект мелкий для запрошенного размера),
`BG_LEFTOVER` (`border_opaque_fraction > 0.02`), `NO_SOFT_EDGE`, `LOW_COVERAGE`, `TINY_OBJECT`, `CLIPPED`
(контент касается края). Каждое предупреждение несёт машинный `fix`-дельту конфига для авто-применения.

**Остальное:** апскейл (`ResizeConfig.upscale='auto|lanczos|edge|ai'`, edge = ×2 Lanczos + unsharp по
Sobel-маске); препроцессинг (`core/enhance.py`: percentile-stretch/gamma/денойз, **условно** — только
если эвристика нашла проблему); палитра (`core/palette.py`: median-cut по непрозрачным пикселям →
dominant + suggest_bg); адаптивный детект фона (кластеризация border-пикселей → multi-key, который
`_color_key` уже поддерживает; опц. градиентная модель).

## B. UX-копайлот (помощник в интерфейсе)

| Фича | Польза | Сложность | Эффект |
|------|--------|:---------:|:------:|
| Структурированные QA-предупреждения вместо сырых notes | Понятный сигнал «хорош ли результат» | M | высокий |
| Кнопка «Исправить» рядом с предупреждением (one-click) | Снимает паралич «что крутить» | S | высокий |
| Авто-рекомендация пресета по картинке (баннер) | Онбординг: подсказка нужного пресета | M | высокий |
| Ручная кисть маски (добавить/убрать области) | Локальная правка вместо глобальных порогов | L | высокий |
| Undo/Redo и история настроек | Возврат к удачному состоянию (Ctrl+Z) | M | средний |
| Слайдер «до/после» + режим «разница» | Видно, что именно убрали | S | средний |
| Ручной кроп-фрейм (перетаскиваемая рамка) | Прямое кадрирование | M | средний |
| Онбординг-тур + «попробовать на примере» | Снижает порог входа, поднимает конверсию | M | средний |
| Доступность (ARIA, клавиатура, фокус) | Скринридеры, клавиатурная навигация, юр. требование | M | средний |

**Кнопка «Исправить».** Каждый `Warning` из `qa.py` несёт `fix` — словарь дельт (`{'background.tolerance':
'+12'}`). В `web/app.js` кнопка применяет дельты к input-ам (clamp по min/max), запускает пересчёт.
**Замкнутая петля:** измерил → предложил → применил → перепроверил; если предупреждение исчезло — кнопка
пропадает сама. Дельта адаптивна (величина `border_opaque_fraction` масштабирует поправку tolerance).

**Ручная кисть маски.** Два режима (стереть фон → alpha=0 / вернуть объект → alpha=255), штрихи
`{x,y,r,mode}` шлются на `POST /api/paint`, применяются в `background.py:apply_paint_strokes`, хранятся
в `Session` как векторные операции (переживают смену ползунков, попадают в экспорт). «Вернуть объект»
ограничивается похожим RGB → магнитная кисть приклеивается к краю.

**Слайдер «до/после».** Полностью клиентский поверх имеющихся data-url (оригинал + foreground). Режим
«разница» (`mix-blend-mode:difference`) автоматически подсвечивает зоны, где удаление что-то изменило —
направляет внимание к вероятным артефактам.

**Остальное:** undo через снапшоты `buildConfig()` (возврат мгновенен благодаря серверному кэшу
сессий); ручной кроп-фрейм (`bbox_override` per object_index); онбординг-тур (`data-tour-step` + `tour.js`,
кнопка «попробовать на примере» через готовый `/api/upload`); доступность (кнопки-объекты с ←/→,
`aria-live` для статуса/ошибок, `:focus-visible`, `prefers-reduced-motion`, `aria-valuetext` на ползунках).

## C. Продвинутая обработка изображения

| Фича | Польза | Сложность | Эффект |
|------|--------|:---------:|:------:|
| Морфологическая постобработка маски (closing/holes/feather) | Чистый край, заделка дыр, без бахромы | M | высокий |
| AI-матирование мягких краёв (волосы/мех/стекло) | Качественный вырез сложных объектов | L | высокий |
| Замена фона (цвет/градиент/блюр/картинка) | Логотип на подложке без Figma | M | высокий |
| Эффекты слоя: тень, скругление, рамка/обводка | Готовый ассет, а не «вырезка» | M | высокий |
| Safe-area паддинг + maskable-проверка | Иконка не обрежется маской на Android 13+/PWA | S | средний |
| AVIF (статичный) | Современный формат, −20-50% веса vs WebP | S | средний |
| Анимированные ассеты (GIF/APNG/animated WebP) по кадрам | Лоадеры, стикеры, анимированные фавиконы | L | средний |
| AI-апскейл (Real-ESRGAN опц. + smart fallback) | Крупный размер из мелкого источника | L | средний |
| Настоящая векторизация (PNG→SVG path) | Масштабируемый SVG для логотипов | XL | средний |

**Морфологическая постобработка.** `_refine_alpha(alpha,cfg)` в конце `_color_key`: бинаризация →
closing (MaxFilter/MinFilter) → fill-holes (flood-fill от рамки, отличает сквозную дыру от внутренней) →
median(3) против бахромы → лёгкий gaussian-feather 1-2px. Радиус адаптивен от размера объекта.

**AI-матирование.** Расширить `_remove_ai`: `remove(im, alpha_matting=True, ...)`; гибридный под-режим
`auto+matte` — color-key для тела, trimap-матирование только на узкой краевой полосе (полоса берётся из
уже посчитанного `ramp ∈ (0,1)` дилатацией). Опц. ONNX, иначе guided-filter; фолбэк без падения.

**Замена фона.** Стадия `stage_compose` (между crop и resize), `core/compose.py`: режимы
`none|color|gradient|blur|image`. Для blur берётся исходный кадр до удаления фона. Цвет по умолчанию
может авто-подбираться контрастным к среднему цвету объекта.

**Эффекты слоя.** `core/effects.py` + стадия `stage_effects` после crop: `drop_shadow` (GaussianBlur
альфы + сдвиг + тонирование), `round_corners` (ImageDraw.rounded_rectangle / суперэллипс-squircle),
`stroke` (дилатация alpha − оригинал). Радиусы/blur в **процентах** от стороны → консистентно от 16 до
1024px. Squircle совпадает с формой iOS.

**Прочее:** safe-area (`CropConfig.safe_area_pct` + `safe_area_report` для круга/центральных 80%); AVIF
(`'avif'` в formats, проверять `features.check('avif')`, иначе note); анимация (`load_frames` +
`run_animated` с заморозкой bbox/key_color по первому кадру, чтобы объект не «прыгал», + `save_all`);
векторизация (`core/vectorize.py`, marching-squares/Douglas-Peucker или vtracer/potrace; авто-решение
векторизуемости по числу цветов, фото → raster-fallback с заметкой).

## Связка фич в «умный» сценарий

```
Загрузка → enhance? (если тусклый) → classify (тип ассета) → рекомендация пресета →
remove_bg (+ matting на сложном) → palette (dominant/theme_color) →
saliency-crop под цель → effects (тень/скругление/safe-area) →
render targets (rect W×H, retina, AVIF) → bundle (манифесты) → QA-отчёт + предупреждения
```

Каждый шаг либо принимает решение сам (с возможностью override), либо предупреждает пользователя —
это и есть «умность»: меньше ползунков, больше готового и проверенного результата.
