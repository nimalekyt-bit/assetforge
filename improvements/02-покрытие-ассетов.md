# 02 — Покрытие типов ассетов

Цель владельца: инструмент должен уметь сделать **почти любой** ассет. Сейчас всё упирается в то, что
экспорт умеет только квадраты и «по ширине». Ниже — чего не хватает и что добавить.

## Чего мы не покрываем (из аудита)

- **[высокая] Соцсети** — есть только Discord. Нет Twitter/X, Instagram, YouTube, TikTok, LinkedIn,
  Telegram, VK, Facebook (точные размеры профилей/обложек/баннеров).
- **[высокая] Email и веб-баннеры/реклама** — IAB-размеры, Google/Facebook Ads.
- **[высокая] App-store скриншоты в рамке устройства** (device-frame mockup).
- **[высокая] Платформенные бандлы/манифесты** — см. [`04-платформенные-выгрузки.md`](04-платформенные-выгрузки.md).
- **[средняя] Игровые texture-атласы** — есть нарезка спрайтов, нет обратной упаковки с картой кадров.
- **[средняя] Печать** — нет мм/DPI/bleed/CMYK/crop-marks.
- **[средняя] Стикеры/эмодзи** — нет точных требований платформ, обводки, лимитов веса, анимации.
- **[средняя] Эффекты** — тень/скругление/рамка/градиент.
- **[низкая] Водяной знак, плагины, AVIF, авто-рекомендация пресета.**

## Что обязан выдавать «полный» инструмент (из ресёрча)

Сводка обязательных выходов, на которые ориентируются конкуренты:

- iOS Xcode-бандл: `AppIcon.appiconset` + валидный `Contents.json`, источник 1024×1024 **без альфы**.
- Android-бандл: `res/mipmap-{mdpi…xxxhdpi}/ic_launcher.png` + Play Store 512 + adaptive
  (`mipmap-anydpi-v26/ic_launcher.xml`, foreground/background, **monochrome** для Android 13+).
- Полный favicon-пакет: мультиразмерный `favicon.ico` + PNG-серия + `apple-touch-icon` 180 + SVG-favicon
  + maskable (192/512, safe-zone 80%) + `site.webmanifest` + **готовый HTML-сниппет** `<link>`-тегов.
- PWA `manifest.json`: name/short_name/start_url/display/colors + icons (≥192 и 512) с `purpose:any` И `maskable`.
- Современные форматы: **AVIF** (вход+выход), улучшенный WebP, опц. lossy-квантизация PNG (−50-70%).
- Точные размеры соцсетей, app-store скриншотов, печати (мм/DPI/bleed), email (фикс-ширина, @2x, непрозрачность).
- Анимированные ассеты: APNG / animated WebP / GIF.
- Авто-определение **типа** входа и авто-рекомендация пресета; QA-предупреждения по качеству.
- Превью в реальном контексте (device-mockup, favicon в табе, Google-сниппет), а не только свет/тёмный/шахматка.
- Эффекты: тень (мягкость/направление), скругление, рамка, safe-area-гайды.

## Предлагаемые фичи покрытия

| Фича | Польза | Сложность | Эффект |
|------|--------|:---------:|:------:|
| Прямоугольные таргеты W×H | Фундамент под все несквадратные ассеты | L | высокий |
| Пресет-пак соцсетей | Десятки точных размеров платформ из одного исходника | M | высокий |
| App-store скриншоты в рамке устройства | Маркетинговые скриншоты для сторов | L | высокий |
| Subject-aware центрирование (fit=cover) | Контент не режется под несквадратную рамку | M | высокий |
| Игровой спрайт-атлас + JSON/CSS-карта | Сборка спрайтов в атлас для движков/веба | L | средний |
| Печатные ассеты: DPI/мм/bleed/CMYK | Визитки, стикеры, флаеры | M | средний |
| Email/маркетинг пресет-пак | Hero/signature/header, @2x, непрозрачность | S | средний |
| Стикеры/эмодзи с die-cut обводкой | Telegram/Discord/эмодзи с лимитами платформ | M | средний |

### Прямоугольные таргеты W×H (фундамент)
- **Проблема:** `ExportConfig.sizes` — плоский `list[int]`, resize/crop только «квадрат или по ширине».
  Без этого ни один несквадратный ассет невозможен — это блокирует всю тему.
- **Решение:** `targets: list[NamedTarget {name,w,h,fit,bg,format}]`, `fit ∈ {pad|cover|contain|stretch}`;
  `render_target` в `resize.py` поверх существующего letterbox; ветка в `build_export`. Обратная совместимость.
- **В чём «ум»:** `fit=cover` центрирует под точную рамку; `bg='auto'` = доминирующий цвет края.
- **Затронуть:** `core/config.py`, `core/resize.py`, `core/export.py`, `core/pipeline.py`, `core/crop.py`.
- **Готово, когда:** пресет с `targets=[{og,1200,630,cover}]` даёт ровно 1200×630 без искажения; старые
  квадратные пресеты не ломаются; тест проверяет размеры и сохранение пропорций.

### Пресет-пак соцсетей
- **Проблема:** под соцсети нужны десятки точных размеров — сейчас каждый делается вручную в Figma.
- **Решение:** JSON-пресеты (`social-og`, `twitter`, `youtube`, `instagram`, `linkedin`, `social-avatars`,
  агрегат `social-all`) со списком `targets` из спецификаций платформ; опц. `scripts/gen_social_presets.py`.
- **В чём «ум»:** каждый таргет несёт правильный `fit` и фон под СВОЮ платформу (аватар→cover+авто-фон,
  баннер с текстом→contain), а не один режим на всё.
- **Готово, когда:** `preset('social-og')` даёт корректно названные файлы точных размеров; пресеты видны
  в CLI/web; пиксельная проверка в тесте.

### App-store скриншоты в рамке устройства
- **Проблема:** для сторов нужны строгие разрешения (iPhone 6.7" 1290×2796, iPad 2048×2732, Android
  1080×1920) и часто кадр устройства. Сейчас разработчик ресайзит и рисует рамки вручную.
- **Решение:** `core/mockup.py` — `frame_in_device(content, device_spec)`; спеки в `assets/devices/*.json`
  (canvas, screen_rect, corner_radius, notch); процедурная рамка через `ImageDraw.rounded_rectangle`
  (без бинарных ассетов для MVP). Пресеты `appstore-screenshots.json`/`playstore-screenshots.json`.
- **В чём «ум»:** контент сам вписывается в реальную экранную область со скруглением/notch — готовый
  маркетинговый скриншот, а не голый ресайз.

### Subject-aware центрирование при fit=cover
- **Проблема:** тупой центр-кроп под несквадратную рамку режет важный контент за край.
- **Решение:** в `render_target` при `fit=cover` вычислять центр обрезки по «центру масс» контента
  (alpha + простая saliency: |dx|+|dy| на numpy), не по геометрии; опц. `focus=[fx,fy]`.
- **В чём «ум»:** рамка наводится на значимую часть кадра, объект остаётся в кадре при любом соотношении.
- *(Полная версия saliency — в [`03-умные-фичи.md`](03-умные-фичи.md).)*

### Игровой спрайт-атлас
- **Проблема:** `detect.py` режет спрайты, но обратной упаковки в атлас с координатной картой нет.
- **Решение:** `core/atlas.py` — `pack_atlas` (shelf-упаковка), `frames`-карта (TexturePacker-hash + CSS);
  источник кадров — `detect.split_objects`; пресет `sprite-atlas.json`, `pow2`-флаг под GPU.
- **В чём «ум»:** разрезал лист спрайтов → упаковал назад в оптимальный атлас; padding против bleeding.

### Печатные ассеты (DPI/мм/bleed/CMYK)
- **Решение:** `NamedTarget` + `unit('px'|'mm'|'in')/dpi/bleed_mm/safe_mm`; `px=round(mm/25.4*dpi)`;
  bleed-расширение холста; запись DPI в PNG; опц. `convert('CMYK')` для TIFF/JPEG. Пресеты `print-*.json`.
- **В чём «ум»:** задаёшь миллиметры — инструмент сам считает пиксели под DPI и добавляет bleed/safe.

### Email/маркетинг
- **Решение:** `email-hero.json` (600/1200@2x), `email-signature.json`, `email-header.json`; непрозрачный
  bg (почтовики не любят alpha); `ExportConfig.retina` дублирует width-таргеты ×2; JPG для веса.

### Стикеры/эмодзи с обводкой
- **Решение:** эффект `outline` в `core/effects.py` (дилатация alpha − маска → кольцо под объектом);
  пресеты `sticker-telegram.json` (512, обводка, ≤512KB), `sticker-discord.json`, `emoji-set.json`;
  авто-контроль веса (`Image.quantize` + предупреждение).
- **В чём «ум»:** die-cut обводка строится из самой alpha-маски, повторяет силуэт любого объекта;
  авто-понижение до палитры под лимит веса платформы.
