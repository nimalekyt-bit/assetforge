# 01 — Аудит текущего функционала

Аудит вёлся по реальному коду (6 агентов, по области каждый). Для каждой области: что уже есть и
полный список гэпов по severity (`критично` → `высокая` → `средняя` → `низкая`). Всего выявлено
**75 гэпов**.

---

## 1. Движок удаления фона и матирование

**Что уже есть:** авто-детект типа фона по рамке (alpha/white/solid/chroma/checker/none); color-key
с двумя порогами (мягкий край); краевая декантаминация (восстановление чистого цвета объекта на
полупрозрачных пикселях); despill для хромакея с взвешиванием по краю; нормализация существующей
alpha; опциональный rembg/u2net; детекция объектов (connected-components, XY-cut, grid); сортировка
в reading-order; тесты по метрикам.

**Гэпы:**

- **[высокая] Нет мягкого матирования (волосы/мех/стекло).** Линейный ramp между порогами не
  восстанавливает alpha на тонких краях → «хвост» фона или просвечивание. Фото с волосами непригодно
  без Photoshop.
- **[высокая] Нет замены/подстановки фона.** Умеем только в прозрачность; нельзя положить логотип на
  белый/градиент/картинку. Для непрозрачных подложек пользователь уходит в Figma.
- **[средняя] Нет AI-апскейла мелких иконок.** Только Lanczos/Bicubic/Nearest → 32px к 512px мылится.
- **[средняя] Нет color-constancy для неравномерного освещения.** Детект по mean/std рамки не тянет
  фото с гуляющим на 20-30 оттенков фоном → авто откатывается к `none`/`solid` с неверным порогом.
- **[средняя] Нет морфологической постобработки маски** (closing/fill-holes/median/feather) → дыры
  внутри фигур, рваный край, бахрома.
- **[средняя] Авто-детект не видит многослойные/полупрозрачные/градиентные фоны** (анализ только рамки 6%).
- **[средняя] Нет режима для полупрозрачных объектов** (тени, стекло) — теряются soft-shadow/отражения.
- **[средняя] Нет subject detection → нет рекомендации пресета.**
- **[средняя] Нет QA-предупреждений о качестве результата.**
- **[низкая] Нет multi-channel масок** (luma-key/similarity-key для спец-случаев).
- **[низкая] Нет препроцессинга входа** (гистограмма/денойз/контраст).
- **[низкая] Нет коррекции цветного гало** (despill только для зелёного/синего хромакея).
- **[низкая] Нет анимации (GIF/APNG/animated WebP), нет AVIF, нет истории/проектов, нет публичного API,
  нет платформенных манифестов** (продублировано в профильных областях ниже).

## 2. Детекция, сегментация и обрезка

**Что уже есть:** bbox по alpha без полупрозрачного шума; connected-components (run-length + union-
find); режимы `single|auto|objects|grid|xycut`; склейка осколков к крупным объектам; reading-order;
дилатация-merge; паддинг %, letterbox в квадрат с align, fit по ширине; кэшируемые стадии пайплайна.

**Гэпы:**

- **[высокая] Нет subject-aware/saliency-кропа под несквадратные цели.** Кроп чисто геометрический;
  под баннер 16:9 / OG 1200×630 контент режется или тонет в паддинге.
- **[высокая] Нет детекции лиц / ROI.** Аватары/портреты кадрируются неверно.
- **[средняя] Нет авто-выравнивания/дескью/перспективы** — кривой логотип экспортируется кривым.
- **[средняя] Слайсинг спрайтов только по равномерной сетке** — нет авто-детекта спрайтов разного размера.
- **[средняя] Нет адаптации под плотности платформ** (iOS @1x/@2x/@3x, Android density buckets, safe-area).
- **[низкая] Нет QA/авто-подсказок** по качеству и стратегии разделения.
- **[низкая] Нет умной группировки** связанных объектов (иконка+подпись) и layout-хинтов.
- **[низкая] Нет авто-определения типа ассета / рекомендации пресета.**

## 3. Экспорт, форматы и платформенные выгрузки

**Что уже есть:** PNG (всегда, опц. optimize), WebP (lossy q90/lossless), мультиразмерный ICO, ICNS,
SVG-обёртка (PNG в SVG), contact-sheet с подписями, ZIP со структурой папок, `manifest.txt`,
расширяемые JSON-пресеты, batch-экспорт, letterbox без искажений, unsharp при даунскейле.

**Гэпы:**

- **[критично] Нет платформенных манифестов** (Xcode `Contents.json`, Android mipmap/adaptive/9-patch,
  PWA `manifest.json` + maskable, Windows MSIX). Без них набор не «компилируется» — главная боль.
- **[высокая] Нет AVIF** (современный формат, baseline 2025-2026).
- **[высокая] Нет системы шаблонов имён файлов** по платформам (Apple/Android/Windows/Web конвенции).
- **[высокая] «SVG» — не векторизация**, а PNG в base64-обёртке: не масштабируется, мутный на Retina.
- **[средняя] Нет анимированных форматов** (APNG/animated WebP/GIF) ни на входе, ни на выходе.
- **[средняя] Нет DPI/цветовых профилей** (sRGB/Display P3/ICC, EXIF/IPTC) — потеря для печати/профи.
- **[средняя] Нет QA-предупреждений об экспорте** (апскейл сверх исходного, обрезка, несоответствие платформе).
- **[средняя] Нет истории/проектов** (persist конфига в БД).
- **[низкая] Нет safe-area/inset** для платформ с notch и maskable.

## 4. Покрытие типов ассетов (что НЕ умеем делать)

**Что уже есть (пресеты):** icon-set, favicon, launcher, web-logo, discord, android, ios,
windows-store, steam, all. Форматы PNG/WebP/ICO/ICNS/SVG-wrapper. Удаление фона, разделение объектов,
обрезка, ресайз, contact-sheet, batch, web/CLI/desktop/SaaS.

**Гэпы (категории ассетов, которые не покрываем):**

- **[высокая] Соцсети — почти не покрыты.** Есть только Discord. Нет Twitter/X (PFP 400×400, header
  1500×500), Instagram (1080×1080/1920), YouTube (thumbnail 1280×720, channel art 2560×1440), TikTok,
  LinkedIn (cover 1584×396), Telegram, VK, Facebook.
- **[высокая] Нет email/веб-баннеров и рекламы** (IAB 728×90/300×250/160×600, Google/Facebook Ads).
- **[высокая] Нет app-store скриншотов в рамке устройства** (device-frame mockup).
- **[высокая] Нет платформенных бандлов/манифестов** (mipmap-структура, Contents.json, PWA manifest).
- **[средняя] Игровые ассеты:** есть grid-нарезка, но нет обратной упаковки в texture atlas с JSON/CSS-картой.
- **[средняя] Печать:** нет мм/DPI/bleed/CMYK/crop-marks (визитки, стикеры, флаеры).
- **[средняя] Стикеры/эмодзи:** нет точных требований платформ, обводки-контура, лимитов веса, анимации.
- **[средняя] AI ограничен только удалением фона** (нет апскейла, saliency, генерации фона, классификации).
- **[средняя] Нет эффектов** (тень/скругление/рамка/градиент).
- **[средняя] Нет фильтров/трансформаций** (яркость/контраст/поворот/нормализация набора).
- **[средняя] Нет истории/проектов, нет публичного API.**
- **[низкая] AVIF, водяной знак (поле есть — функции нет), система плагинов, авто-рекомендация пресета.**

## 5. UX веб/CLI/desktop и интеллектуальная помощь

**Что уже есть:** авто-детект фона, двухпороговый мягкий край + despill, умная обрезка, разделение
объектов, ресайз с unsharp, экспорт + contact-sheet, расширяемые пресеты, web-SPA с тремя стадиями
превью (фон/обрезка/реальные размеры), 12 реактивных контролов с дебаунсом, batch drag-drop, CLI с 14
опциями, desktop, кэш сессий, русский интерфейс, человекочитаемые ошибки + апселл, стадийный пайплайн.

**Гэпы:**

- **[высокая] Нет QA-предупреждений** (мелко для размера, контент у края, низкий контраст, проблемы с alpha).
- **[средняя] Нет авто-рекомендации пресета по картинке.**
- **[средняя] Нет undo/истории и сравнения «до/после».**
- **[средняя] Нет ручного редактирования маски/кропа.**
- **[средняя] Нет встроенного AI для детекции типа содержимого.**
- **[средняя] Нет AVIF/анимации/платформенных манифестов** (профильно — см. область 3).
- **[низкая] Нет доступности** (ARIA, клавиатура, фокус, скринридеры).
- **[низкая] Нет онбординга/тура** для новичков.
- **[низкая] Нет авто-рекомендации оптимального размера.**
- **[низкая] Нет встроенной справки/тултипов в UI.**
- **[низкая] Нет логирования/диагностики ошибок.**

## 6. SaaS, коммерция и удержание

**Что уже есть:** регистрация/вход с верификацией e-mail, cookie-сессии (SameSite strict), тарифы
Free/Pro/Business, помесячные квоты с атомарным списанием, платежи Stripe/ЮKassa/manual + промокоды,
админка, личный кабинет, статические страницы, сброс пароля, API-эндпоинты с auth+квотами, кэш сессий,
SQLAlchemy-модели (User/Payment/UsageRecord/PromoCode).

**Гэпы:**

- **[высокая] Нет истории экспортов / сохранённых проектов** — работа эфемерна, всё теряется с сессией.
- **[высокая] Нет командных функций/воркспейсов** — Business позиционируется «для студий», но это
  одиночный аккаунт (нет инвайтов, общих квот, ролей).
- **[высокая] Нет публичных API-ключей** — только cookie-сессия, невозможна M2M-интеграция/CI.
- **[средняя] Нет сторонних интеграций** (Figma, Slack, webhook, npm).
- **[средняя] Нет триал-периода** (Free — постоянный, без trial→paid конверсии).
- **[средняя] Нет онбординга/feature discovery.**
- **[низкая] Нет пользовательских/избранных пресетов.**
- **[низкая] Нет очереди/async batch-обработки.**
- **[низкая] Нет аналитики использования** (по пресетам/форматам/размерам).
- **[низкая] Водяной знак** — поле в plans.json есть, enforcement нет.
- **[низкая] Нет кастомизации/брендинга contact-sheet.**
