# 09 — Аудит SaaS / Desktop и рекомендации

> Дата: 2026-06-14. Разбор после реализации SaaS-слоя, desktop-приложения и админ-панели.
> Предыдущие файлы (01–07) — про продуктовый roadmap движка; этот — про монетизацию,
> прод-готовность, безопасность и масштаб. Что реально отделяет проект от «работающего бизнеса».

## Текущее состояние (кратко)
Функционально проект богатый: единый движок (веб/desktop/CLI), self-installing desktop-`.exe`
с авто-обновлением и фримиум-входом, SaaS-сайт (аккаунты, 2FA, тарифы, промокоды, реферал,
кабинет), большая админка (дашборд с графиками, пользователи с сегментами/массовыми действиями/
таймлайном, платежи, тарифы из UI, релизы десктопа, A/B-флаги, когорты, вебхуки, журнал входов,
аудит, логи, здоровье, RBAC, impersonation, IP-allowlist/blocklist, force-logout), публичный
`/api/v1`, инфраструктура (Alembic, опц. Redis, логирование, CSRF, лимиты загрузки). Тесты: 65 зелёных.

## КРИТИЧНО (без этого нельзя запускаться/зарабатывать)

### 1. Реальные платежи (ЮKassa) — приоритет №1
Биллинг на заглушках (`stripe_provider`/`yookassa_provider` бросают `RuntimeError`), активен только
`manual`-демо. Купить Pro за деньги нельзя → метрики выручки, чеки, возвраты, реферальные награды —
декорации. Закрыть: подключить ЮKassa (создание платежа + проверка вебхука по подписи), показать
фискальный чек (54-ФЗ) в кабинете.

### 2. Деплой и прод-готовность
`DEPLOY.md` — инструкция «на будущее», реально не деплоилось. Риски:
- Нет Docker-образа в работе / CI (хотя бы прогон тестов).
- **Секреты в дереве проекта** (`.env`: пароль Supabase, `ASSETFORGE_SECRET`) — перед публичным
  деплоем ротировать, держать только в окружении.
- Обновления десктопа и логин по HTTP — для интернета нужны **HTTPS + подпись/чексумма** exe.
- Exe не подписан → SmartScreen/антивирусы (нужен code-signing сертификат).

### 3. Синхронная тяжёлая обработка
Удаление фона/нарезка (особенно AI-rembg) идёт в обработчике запроса → под нагрузкой забивает
воркеры. Вынести тяжёлые экспорты в очередь (RQ/Celery или пул процессов), ограничить конкуренцию,
не держать большие картинки в RAM.

## ВАЖНО (надёжность и масштаб)

### 4. Точечный security-хардненинг
- Нет CSP/security-заголовков; Chart.js с CDN без SRI (запинить хэш или self-host).
- У API-ключей нет скоупов и индивидуального rate-limit.
- Нет блокировки брутфорса сверх rate-limit; загрузки не сканируются антивирусом.
- Клиентские лимиты десктопа обходимы (осознанный компромисс).

### 5. Интеграционные тесты
Сейчас юнит-тесты + рендер-200 через `dependency_overrides` (минуя реальную авторизацию/CSRF/сессии).
Добавить сквозные HTTP-сценарии (регистрация→оплата→экспорт с настоящими cookie+CSRF), тест
IP-middleware, тест автообновления десктопа против поднятого сервера в CI.

### 6. Наблюдаемость
Есть логи (кольцевой буфер + опц. Sentry). Нет: healthcheck-эндпоинта для балансировщика, метрик
(Prometheus), структурного request-лога, планировщика для `tasks.py reminders` (нужен внешний cron).

## ПРОДУКТ (рост)

### 7. Интеграции — где деньги Pro/Business
- **Плагин Figma / расширение браузера** — нарезка прямо из дизайна (киллер-фича).
- **Telegram-бот** на нашем `/api/v1`.
- **Публичные API-доки** (FastAPI `/docs` обернуть) + примеры curl.
- **Исходящие вебхуки**, Zapier/Make.

### 8. Усиление инструмента (удержание)
- Ручная доводка маски (кисть «вернуть/стереть»).
- История экспортов с повторным скачиванием (нужно хранилище: S3/диск + TTL).
- Командные аккаунты (seats) для Business.
- Онбординг (welcome-тур, демо «попробовать без загрузки»).
- Мобильная адаптация инструмента.

## КОД / ПОДДЕРЖКА
- `saas/routes.py` разросся — разбить на роутеры по областям (`auth`/`account`/`billing`/`admin`/`desktop`).
- Автогенерация Alembic пару раз добавляла лишний self-FK — миграции держать под ревью.
- Роуты дёргают приватные функции биллинга (`_extend_plan`) — вынести в публичный сервис.
- RU-only — заложить i18n при выходе на зарубеж.

## Рекомендованный порядок
1. **ЮKassa + чеки** — открывает деньги и делает метрики реальными.
2. **Деплой-пакет**: Docker + gunicorn/uvicorn за nginx/HTTPS, ротация секретов, CI с тестами.
3. **Очередь для тяжёлой обработки** + лимит конкуренции.
4. **Security**: заголовки/CSP, подпись обновлений и code-signing exe, SRI для CDN.
5. **Интеграция-флагман** (Figma-плагин или Telegram-бот).
6. Интеграционные тесты + healthcheck/метрики.

**Вывод:** продукт функционально богат, но **не монетизируется и не задеплоен** — это две вещи,
которые отделяют его от работающего бизнеса. Вкладываться сначала в них, потом в новые фичи.
