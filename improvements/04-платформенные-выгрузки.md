# 04 — Платформенные выгрузки и манифесты

Чтобы «делать всё», мало нарезать размеры — нужно отдавать готовые **бандлы** под платформы с
правильной структурой папок, манифестами и именами файлов. Это та самая «последняя миля», ради которой
люди ищут платный инструмент (appicon.co, RealFaviconGenerator). Сейчас инструмент отдаёт голые PNG, и
разработчик вручную раскладывает их по папкам и пишет конфиги — это **критичный** гэп (severity:critical
в аудите экспорта).

Хорошая новость: мы **уже** генерим нужные PNG/ICO размеры. Не хватает только обёртки — а манифесты —
это шаблоны JSON/XML + правильные имена файлов, **не алгоритмы**. `package_zip`/`write_to_dir` уже
корректно сохраняют вложенные папки из имён артефактов, так что серверный/CLI слой почти не меняется.

## Обязательные выходы по платформам (из ресёрча)

### iOS / macOS
- App Icon: одиночный PNG **1024×1024, без альфа-канала, без скруглений** (маску накладывает система),
  sRGB/Display P3, квадрат full-bleed.
- Asset Catalog: папка `AppIcon.appiconset/` + `Contents.json` (modern: idiom=universal, platform=ios,
  size=1024x1024; опц. легаси-набор 20/29/40/60/76/83.5pt @1x/2x/3x + 1024 marketing). **83.5pt → 167px.**
- macOS: 16/32/128/256/512 @1x и @2x (до 1024), **с альфой и собственным скруглением**, как `.icns`.

### Android
- Adaptive: foreground + background 108dp по плотностям (108/162/216/324/432px для mdpi…xxxhdpi); весь
  ключевой контент в центральной safe-zone (видимый круг ~66dp).
- `mipmap-anydpi-v26/ic_launcher.xml` с `<background>`, `<foreground>`, **`<monochrome>`** (силуэт для
  Material You, Android 13+).
- Legacy: `mipmap-{mdpi 48, hdpi 72, xhdpi 96, xxhdpi 144, xxxhdpi 192}/ic_launcher.png` (+ круглый).
- Play Store: 512×512, 32-бит PNG, полный квадрат.

### Web / PWA / favicon
- `favicon.ico` (16/32/48), `favicon-16/32/48.png`, `icon-192/512.png`, `apple-touch-icon.png` 180
  (непрозрачный фон), SVG-favicon.
- `site.webmanifest`: name/short_name/start_url/display/background_color/theme_color + icons ≥192 и 512,
  записи `purpose:any` И `purpose:maskable`.
- **Maskable-иконка**: значимый контент в центральных 80% (радиус 40%), чтобы маска ОС не обрезала.
- **Готовый HTML-сниппет** в `<head>`: `<link rel=icon>`, `apple-touch-icon`, `<link rel=manifest>`,
  `<meta name=theme-color>` (+ legacy `msapplication-*`).

### Windows
- MSIX/Store tiles: `Square44x44Logo.scale-200.png`, `Square150x150Logo.*`, `Wide310x150Logo.*`,
  `StoreLogo.png`, targetsize-варианты (16/24/32/48/256), `browserconfig.xml`/AppxManifest-фрагмент.
- `.ico`: канонический набор 16/24/32/48/64/128/256 (256 — PNG-compressed внутри ICO).

### В каждый ZIP
- Сводный README/manifest: какой файл куда положить (favicon → корень, `.appiconset` → Xcode,
  `res/mipmap-*` → Android-проект), чтобы пользователь не гадал.

## Предлагаемые фичи

| Фича | Польза | Сложность | Эффект |
|------|--------|:---------:|:------:|
| Слой «bundle»-эмиттеров поверх ExportResult | Архитектурный фундамент бандлов | M | высокий |
| Xcode `AppIcon.appiconset` + `Contents.json` | Drag-and-drop в Xcode без ручной сборки | M | высокий |
| Android adaptive icons (mipmap + XML + слои) | Готовая `res/`-структура с safe-zone | L | высокий |
| PWA `manifest.json` + maskable | Устанавливаемое PWA из коробки | M | высокий |
| Полный favicon-набор + HTML-сниппет | Копипаст-теги без гугления | M | высокий |
| Windows MSIX / Store tiles | UWP-ассеты с scale-именами | L | средний |
| Декларативные бандлы в JSON-пресетах | Бандл как данные, без правки Python | S | средний |
| Выбор/предпросмотр бандлов в web-UI и CLI | Дерево будущих файлов до скачивания | M | средний |

### Слой «bundle»-эмиттеров (фундамент)
- **Решение:** `core/bundles.py` — реестр `BUNDLE_EMITTERS: dict[str, Callable]`. Каждый эмиттер получает
  уже отрендеренные размеры (`size→RenderedSize`) и возвращает `list[ExportArtifact]` с готовыми
  путями/именами (`kind='bundle-asset'|'manifest'`). `build_export` после рендера зовёт эмиттеры для
  `ecfg.bundles`. Имена-пути кладутся прямо в `ExportArtifact.name` — ZIP/диск уже сохранят папки.
- **В чём «ум»:** эмиттер сам добирает недостающие размеры из `base` (`resize_to`) и дедуплицирует уже
  посчитанные — один источник истины (base RGBA) → любая платформенная раскладка.

### Xcode AppIcon.appiconset
- **Решение:** эмиттер `ios_appiconset()`: для каждого (point, scale) из канонической таблицы рендерит
  `pixel=round(point*scale)`, имя `Icon-App-{pt}x{pt}@{scale}x.png`, **флэттит альфу** на непрозрачный фон;
  собирает `Contents.json` (`json.dumps`) как `manifest`-артефакт.
- **В чём «ум»:** учитывает дробный 83.5pt (167px), различает iPhone/iPad/marketing idiom, **сам убирает
  альфу только для iOS** (App Store отклоняет прозрачные иконки) — пользователю не надо знать это правило.

### Android adaptive
- **Решение:** `android_adaptive()`: legacy PNG по density-бакетам; foreground 108dp с контентом в
  центральных 66/108 (safe-zone); background-слой; `mipmap-anydpi-v26/ic_launcher.xml`; Play Store 512.
- **В чём «ум»:** сам вычисляет safe-zone inset и масштабирует контент так, чтобы он пережил любую маску
  (круг Pixel, squircle Samsung); плотности считаются от dp.

### PWA / favicon
- **PWA `pwa()`:** icons (72…512) `any` + maskable-варианты (контент в 80%, непрозрачный фон) +
  `manifest.json`; `theme_color` из доминирующего цвета объекта.
- **Favicon `favicon()`:** `favicon.ico` + PNG-серия + `apple-touch-icon` 180 + `favicon.svg` +
  `site.webmanifest` + `head-snippet.html` со ВСЕМИ корректными тегами. Сниппет генерируется из реально
  созданных файлов (один источник правды → нет висячих ссылок/404).

### Прочее
- **Windows MSIX:** `msix()` — scale-суффиксы (`Square150x150Logo.scale-200.png`), Wide-плитка (несквадратный
  fit по высоте), targetsize unplated, `AppxManifest.snippet.xml`.
- **Декларативные бандлы:** `ExportConfig.bundles: list[str]` + `bundle_options: dict` (bg_color, app_name,
  theme_color, safe_zone_pct). Владелец добавляет `presets/ios-dark.json` с `bg_color='#000'` без Python.
  Неизвестное имя бандла → warning в meta, не падение.
- **UI/CLI:** `/api/presets` отдаёт `bundles` (бейджи); чекбоксы бандлов в `app.js`; новый
  `/api/bundle-tree` (зовёт `build_export`, отдаёт только имена артефактов деревом — превью структуры до
  скачивания); CLI-флаг `--bundles ios,android,pwa`.

> **Совет по очередности (из проверки):** начать с **PWA** (1 JSON + 3 PNG + HTML — самый дешёвый,
> проверяется Lighthouse), затем **favicon** и **iOS** (`Contents.json` тривиален), и в последнюю
> очередь **Android adaptive** (двуслойная foreground/background структура + monochrome — самая сложная,
> требует выделения силуэта).
