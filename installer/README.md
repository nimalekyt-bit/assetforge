# AssetForge — desktop-дистрибутив (установщик + авто-обновление)

Один файл `dist\AssetForge.exe` — это и установщик, и приложение. Пользователю
не нужны ни Python, ни библиотеки, ни права администратора.

## Сборка

```bat
build_exe.bat
```

Результат: `dist\AssetForge.exe`. Его и раздаём пользователям.

## Что происходит у пользователя

1. **Первый запуск** (файл скачан в «Загрузки») — открывается брендированное окно
   установки. По кнопке «Установить» приложение копируется в
   `%LOCALAPPDATA%\AssetForge`, создаются ярлыки (меню Пуск + рабочий стол),
   регистрируется удаление в «Программах и компонентах», при отсутствии
   ставится Edge WebView2 Runtime. Затем запускается само приложение.
2. **Последующие запуски** (через ярлык) — короткая тихая проверка обновлений
   (брендированный сплэш). Есть новее → скачивает и ставит; нет/офлайн → сразу
   открывает приложение в нативном окне.
3. **Удаление** — через «Программы и компоненты» (или `AssetForge.exe --uninstall`):
   удаляются ярлыки, запись реестра и папка установки.

Флаги: `--uninstall`, `--install-silent`, `--no-update`.

## Авто-обновление: раздача с сайта (уже встроено)

Сайт SaaS сам отдаёт дистрибутив и манифест — отдельный хостинг не нужен:

| Маршрут | Что отдаёт |
|---|---|
| `GET /desktop` | страница скачивания в фирменном стиле |
| `GET /desktop/latest.json` | манифест версии (для авто-обновления) |
| `GET /desktop/AssetForge-Setup.exe` | сам установщик (текущий релиз или `dist\AssetForge.exe`) |

Текущий релиз выбирается из БД (управляется в админке **/admin/releases**); если релизов нет —
фолбэк на собранный `dist\AssetForge.exe` и `__version__`. Приложение при проверке шлёт свою версию
(`?from=`), сервер ведёт **статистику версий** (видно в /admin/releases).

### Текущая (локальная) настройка

- `run_saas.py` поднимает сайт на `0.0.0.0:8000` → доступен по локальному IP.
- В `.env`: `ASSETFORGE_BASE_URL=http://192.168.0.105:8000`.
- В `assetforge/update.py`: `DEFAULT_MANIFEST_URL=http://192.168.0.105:8000/desktop/latest.json`.

Если локальный IP изменится — поправьте оба места (или задайте `ASSETFORGE_UPDATE_URL`
на машинах с приложением и `ASSETFORGE_BASE_URL` на сервере). При выходе в интернет
замените локальный IP на ваш домен (желательно по HTTPS).

### Как выкатить новую версию

**Вариант A (через админку, без передеплоя):**
1. Поднимите `__version__`, соберите `build_exe.bat` → `dist\AssetForge.exe`.
2. В админке **/admin/releases** загрузите файл, укажите версию и заметки → он станет текущим.
3. Установленные приложения при следующем запуске увидят новую версию и обновятся.

**Вариант B (без админки):** положите новый `dist\AssetForge.exe`, перезапустите `run_saas.py` —
сработает фолбэк по `__version__`.

Приложение сравнивает версии посегментно (числами) и качает обновление только если
на сервере строго новее.

## Фирменная графика

`installer/make_assets.py` генерирует `AssetForge.ico` и `logo.png` из бренд-палитры
«кузницы» (уголь + раскалённые угли + латунь). Запускается автоматически из `build_exe.bat`.

## Аккаунты и лимиты в приложении (фримиум, Вариант C)

Десктоп работает локально, но тариф берёт из облачного аккаунта (общего с сайтом):

| Режим | Как включается | Лимиты |
|---|---|---|
| **Гость** | без входа | 256px, только PNG, без batch, без AI-фона |
| **Free** | вход в аккаунт | 1024px, PNG/ICO/WebP, batch до 3, без AI |
| **Pro / Business** | вход + оплаченный тариф | до 4096–8192px, все форматы, AI-фон, batch 100+ |

- Вход — в приложении (кнопка «Войти» в шапке) по email+паролю; идёт в облако
  `POST /api/desktop/auth`, возвращается подписанный токен + тариф.
- Тариф валидируется при каждом запуске (`/api/desktop/validate`). Офлайн Pro
  действует `OFFLINE_GRACE_DAYS = 7` дней, затем до повторной онлайн-проверки права
  падают до гостевых.
- Лимиты проверяются локально при экспорте (`entitlement.enforce`). При превышении —
  понятный апселл: гостю «Войти», free-юзеру «Перейти на Pro» (тарифы открываются в браузере).
- Сессия хранится в `%LOCALAPPDATA%/AssetForge/session.json` и удаляется при деинсталляции.

### Адрес облака

Приложение ходит за аккаунтом на `ASSETFORGE_CLOUD_URL` (по умолчанию
`http://192.168.0.105:8000`, задано в `assetforge/cloud.py`). Меняется так же, как и
адрес обновлений: правка дефолта перед сборкой либо переменная окружения
`ASSETFORGE_CLOUD_URL`. При выходе в интернет — заменить на ваш домен (HTTPS).
